# 理解对象

  1. 对象属性的类型

    1. 数据属性

      通过object.defineProperty(对象，属性名，属性的描述符包含{configurable，enumerable，writable，value})

      Object.defineProperty(person,"name",{
    writable:false,
    value:"lily"
})

    2. 访问器属性

      访问器属性不包含数据值，但包含一个获取函数get和一个设置函数set

      Object.defineProperty(person,age,{
    get(){
        return this.age_
    },
    set(newAge){
        if(newAge>this.age_){
            this.age_=newAge
        }
    }
})

      如果要同时编辑多个属性，那么就将Object.defineProperty的第二第三个参数写成{}map

  2. 合并对象

    如果有同名属性，那么后面的覆盖前面。

    let dest={}
let scr={id:"src"}
let result = Object.assign(dest,src)//将后面的obj加入第一个obj
result={...dest,...src};

  3. 对象解构

    let {name:"lily",age: 18} =person

# 创建对象

  1. 基本

    let persoon1 = new Object()
let person = {
    name : "lily",
    age_: 18,
    sayName(){
        console.log(name)
    }
}

//es6新增的一种写法
let name = "a";
let obj = {
    [name+"bc"]:"lily"
};
console.log(obj);//{abc:lily}

  2. 构造函数

    //构造函数
function Person(name,age){
    this.name = name;
    this.age = age;
    this.sayName= function(){
        console.log(this.name);
    };
}
let person2 = new Person("lily",18)
console.log(person2 instanceof Object); //true

    构造函数没有return

  3. 原型模式

    let Person = function (){}
Person.prototype.name = {name:"lily"} 
//这些属性都被添加到了Person的原型prototype属性上去，所以由此创建的新对象实际上是所有实例共享这些原型定义的属性和方法
let person1 = new Person()
let person2 = new Person()
console.log(person1.name===person2.name)//true

    理解原型：每个函数都有一个prototype属性，这个属性是一个对象，这个对象包含一些属性和一个构造函数。在原型上定义的属性和方法可以被对象实例共享。每个构造函数都有一个原型对象，原型都有一个属性指向构造函数，每个实例都有一个内部至深圳执行原型。

    同时，每个对象也可以拥有自己的属性。原型层级：先访问实例的属性，然后访问其原型的属性，有点类似于继承的概念。

  4. 对象的迭代

    关于对象的迭代，可以使用for-in或者object.entries()等方法，会迭代所有的属性，包括实例本身的和其原型的。

# 继承

JavaScript的继承是通过原型链实现的。基本思想是通过原型继承多个引用类型的属性和方法。具体来说，就是一个实例的原型是另一个实例。具体实现方法：

//父类的构造函数
function SuperType(){
    this.property = true;
}
SuperType.prototype.getSuperValue = function(){
    return this.property;
};
//子类的构造函数
function SubType(){
    this.subProperty = false
}
//将子类的原型设为父类实例
SubType.prototype = new SuperType();
SubType.prototype.getSubValue = function(){
    return this.subProperty;
};
let sb = new SubType()
console.log(sb.getSuperValue())//true

如何确定子类与父类？子类 instanceof 父类 =>true

子类方法覆盖：

SubType.prototype.getSuperValue = function (){
    return false;
};//直接用同名的方法覆盖
/*注意不要使用字面量重写。例如：
SubType.prototype = {
	getSuperValue(){
		return false;
	},
	...
};
这样会使后来的这个对象完全覆盖原先的原型对象，不能实现继承。*/

原型链存在的问题：由于多个实例对象的原型为同一个，即所有实例共享原型的属性，如果一个实例改变这个原型（因为继承，所有这个原型本质上也是一个实例）的属性，那么这个改变也会在所有实例中共享。

# 类

  1. 类的定义

    let say = Symbol('say');
class Person{
    constructor(name,age){
        this.name = name;
        this.age = age;
    }
    [say](){
        console.log(this.name,this.age);
    }
    static method(){}
}
let p = new Person();
console.log(p.__proto__===Person.prototype); //true
console.log(p[say]());

    注意：

    1. 每一个实例都对应一个独立的成员对象，不会在原型上共享
    2. 类的构造器外的属性和函数都会定义在类的原型上。

  2. 继承

    1. 语法：class B extends A{...}
    2. 没什么好说的，extends，super，重写和Java没什么大的区别。注意super只能在构造函数和静态方法中使用。

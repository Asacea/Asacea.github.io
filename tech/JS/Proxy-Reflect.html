# 代理Proxy与反射Reflect

# Proxy基础

  1. 理解代理

    在proxy出现之前，我们想要捕获到对象的某个属性发生变化，需要在defineProperty方法的时候，添加get和set捕获器：

    let obj = {};
Object.defineProperty(obj,"data",{
    get(){
        console.log("get:");
    },
    set(){
        console.log("set:");
    }
});

    这种方式存在的问题是，它只能拦截一个属性，并且只能拦截对象，不能拦截其他比如数组。

    Proxy的作用是在对象和对象的属性之间设置一个代理，通过这个代理来实现拦截和监听。它可以拦截目标对象所有的属性。

  2. 创建代理

    Proxy构造函数接收2个参数：目标对象和处理程序对象。

    const target = {
    id : "target"
};
const handler = {
    get(target,property){
        console.log("get:",target,property);
        return target[property];
    },
    set(target,property,value){
        console.log("set:",target,property,value);
        if(property=="num"){
            let list = document.getElementById("list");
            list.innerHTML = value;
        }
        target[property]=value;
    }
};
const proxy = new Proxy(target,handler);
console.log(proxy.id);//GET

  3. 拦截的类型：数组、set、map

    在这些数据结构中，如果访问的属性是方法的话，需要修改this的指向，因为我们希望this指向一个实例，但是在使用代理的时候，实际上是指向了这个代理对象，所以需要手动修改。

    let s = new Set();
let proxy = new Proxy(s,{
    get(target,property){
        console.log("Get");
        let value = target[property];
        if(value instanceof Function){
            return value.bind(target);
        }
        return target[property];
    },
    set(target,property,value){
        console.log("Set");
    }
})
proxy.add(1);

# Reflect

  1. Reflect可以获取目标对象的默认行为，与Proxy对应。

  2. Reflect的作用：

    1. 代替Object的某些方法

      原来：
let obj = {};
Reflect.defineProperty(obj,"name",{
    value:"lily",
    writable:false
});//有屁用？？？

    2. 修改某些Object方法的返回结果

      //很优雅地获得操作地返回值是true or false
if(Reflect.defineProperty(target,property,attributes)){
    //success
}else{ }

    3. 把一些丑陋的命令式变成函数行为

      console.log(Reflect.has(target,property));
//以前的丑陋写法：
console.log(property in target);

    4. 与Proxy配合

      拦截数组

      let arr = [1,2,3,4];
let proxy = new Proxy(arr,{
    get(target,key){
        console.log("get",key);
        return Reflect.get(...arguments);
    },
    set(target,key,value){
        console.log("set",key);
        return Reflect.set(...arguments);
    }
});

      ！！注意对arr和set的不同，set的方法需要重新定义this的指向，原因是：

      对于 Array 对象的监听与 Set 对象的监听之所以行为不同，是因为它们的底层实现不同。在 JavaScript 中，Array 是一个原生对象，而 Set 是一个内置对象。这两者的底层实现和行为是不同的，导致了监听行为不同。

      当你在代理对象上调用 proxy1.push(2) 时，这个操作会被正确地代理到 Array 对象的 push 方法上，因为 push 方法是 Array 对象上的一个可枚举属性。

      而对于 Set 对象，它的 add 方法是内置的，不可枚举。在代理对象上尝试代理 Set.prototype.add 会导致错误，因为代理不支持这种操作。

      所以，不同对象的可枚举属性和内置方法会导致监听行为的不同。你可以考虑在代理时处理这些细节，或者直接代理整个对象以获取更一致的行为。
